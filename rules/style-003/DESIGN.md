## 设计原理

### 为什么这条规则重要

混用 `-` 和 `*` 作为列表标记会导致样式不一致。虽然两者在 Markdown 中功能相同，但保持一致性能提升可读性。

### 示例

```
# ✅ 正确 (统一使用 -)
- 列表项 1
- 列表项 2

# ✅ 正确 (统一使用 *)
* 列表项 1
* 列表项 2

# ❌ 错误 (混用)
- 列表项 1
* 列表项 2
```

---

## 实现逻辑

### 检测方法

```
1. 查找所有列表项行 (/^[\s]*[-*]\s/)
2. 提取使用的标记符号
3. 检查是否同时使用了 - 和 *
4. 如果混用，报告 Hint
```
