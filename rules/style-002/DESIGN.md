## 设计原理

### 为什么这条规则重要

行尾空格是无意义的噪音，在版本控制中会造成不必要的 diff，影响代码审查体验。清理行尾空格是基本的代码整洁习惯。

### 示例

```
# ✅ 正确
const name = "skill";

# ❌ 错误
const name = "skill";␣␣
```

---

## 实现逻辑

### 检测方法

```
1. 逐行检查内容
2. 使用正则 /\s+$/ 匹配行尾空格
3. 报告所有包含尾随空格的行号
```
